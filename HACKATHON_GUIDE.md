# VibePilot 해커톤 실행 가이드

이 문서는 [PRD.md](PRD.md)에서 분리한 해커톤 실행 관련 내용을 정리합니다. PRD는 제품의 문제, 가치, 기능, Copilot SDK 경험을 중심으로 유지하고, 이 문서는 Azure 방향, 4시간 빌드 플랜, 데모 흐름, 제출 준비를 다룹니다.

## 1. Azure AI와 클라우드 방향

VibePilot의 클라우드 이야기는 제품 호스팅과 AI 레이어를 Azure 위에서 운영하는 데 초점을 둡니다.

선호 아키텍처:

- Next.js 웹앱은 Azure Container Apps 또는 Azure App Service에 호스팅한다.
- Copilot SDK 오케스트레이션은 서버 사이드 라우트 또는 별도 API 서비스에서 처리한다.
- AI 모델 레이어는 Azure OpenAI 또는 Microsoft Foundry를 활용한다.
- Azure 호스팅 환경에서는 Managed Identity 기반 인증을 사용한다.
- 소스 코드에 두기 어려운 설정은 Azure Key Vault에 보관한다.
- Application Insights로 요청 시간과 사용자에게 보이는 AI 오류를 가볍게 관측한다.

이 방향에서 Azure는 단순 정적 호스팅이 아니라, 핵심 AI 코칭 루프와 운영 가능한 인증 구조를 받쳐주는 의미 있는 기반이 됩니다.

## 2. 4시간 해커톤 빌드 플랜

| 시간 | 초점 | 결과 |
|---|---|---|
| 0:00-0:30 | 제품 명확화 | PRD와 README를 앱 스토리에 맞춘다. |
| 0:30-1:30 | Copilot SDK 경로 | 구조화된 프로젝트 맥락을 받는 서버 사이드 AI 라우트. |
| 1:30-2:15 | AI 워크플로 UI | 대시보드 안의 스트리밍 로드맵 또는 다음 행동 패널. |
| 2:15-3:00 | Azure 이야기 | Azure OpenAI 또는 Foundry 설정과 Azure 배포 경로. |
| 3:00-3:30 | 신뢰 요소 다듬기 | 친절한 오류 상태, 검토 가능한 제안, 시크릿 비노출 흐름. |
| 3:30-4:00 | 제출 전 정리 | 빌드 확인, 배포 URL 확인, PRD/README 최종 정렬. |

## 3. 데모 시나리오

심사에서 보여주기 좋은 흐름은 다음과 같습니다.

1. `intermediate`를 선택한다.
2. 프로젝트 이름으로 `포트폴리오 피드백 앱`을 입력한다.
3. 목표로 `피드백을 수집하고 액션 아이템을 요약하는 작은 Azure 호스팅 앱을 배포한다`를 입력한다.
4. 프로젝트를 시작하고 생성된 로드맵을 확인한다.
5. VibePilot Coach에게 로드맵 개선을 요청한다.
6. AI가 상태 진단, 다음 행동, 성공 기준을 스트리밍하는 모습을 확인한다.
7. 제안된 체크리스트 변경을 적용하거나 무시한다.
8. 작업 하나를 완료하고 진행률 변화를 확인한다.
9. 최종 제출 전 프로젝트 준비도 리뷰를 실행하고 제출 준비 가이드와 대조한다.

## 4. 제출 준비 가이드

최종 제출에 가까운 앱은 다음 모습을 보여주면 좋습니다.

- Azure에 호스팅된 작동하는 웹앱.
- 눈에 보이는 Copilot SDK 기반 AI 안내.
- AI 흐름에 전달되는 구조화된 프로젝트 맥락.
- 하나 이상의 AI 상호작용에서 보이는 스트리밍 출력.
- AI/모델 레이어로 Azure OpenAI 또는 Microsoft Foundry 활용.
- 명확한 진행 추적과 다음 행동 안내.
- 사용자의 주도권을 유지하는 검토 가능한 AI 제안.
- 배포된 앱 동작과 일치하는 PRD 및 README.
