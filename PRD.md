# VibePilot PRD

## 1. 제품 요약

**제품명:** VibePilot

**한 줄 소개:** VibePilot은 막연한 바이브 코딩 아이디어를 명확하고 추적 가능한 배포 지향 프로젝트 여정으로 바꿔주는 Copilot 기반 개인 생산성 웹앱입니다.

**대상 사용자:** AI 코딩 도구를 활용해 웹앱을 만드는 1인 개발자, 해커톤 참가자, 초보 빌더.

**제출 초점:** VibePilot은 AI 심사 에이전트가 PRD와 실제 실행 앱을 함께 살펴보는 개인 생산성 해커톤을 염두에 두고 설계되었습니다. 제품은 의미 있는 Copilot SDK 워크플로, Azure 기반 AI 활용, 실질적인 생산성 향상, 매끄러운 사용자 경험을 중심에 둡니다.

## 2. 제품 비전

AI 코딩 도구는 강력하지만, 많은 1인 개발자는 아이디어, 구현, 테스트, 배포 사이에서 방향을 잃기 쉽습니다. VibePilot은 이런 사용자를 위한 가벼운 프로젝트 조종석입니다. 다음 할 일을 분명히 보여주고, 진행 상황을 눈에 보이게 만들며, AI를 통해 기획 부담을 낮춥니다.

VibePilot의 목표는 개발자를 대체하는 것이 아닙니다. 개발자가 현재 위치를 파악하고, 더 나은 결정을 내리고, 아이디어에서 배포 가능한 앱까지 꾸준히 이동하도록 돕는 것입니다.

## 3. 문제 적합성

VibePilot은 하나의 구체적인 생산성 문제에 집중합니다. **AI 보조 개발자는 코드를 빠르게 만들 수 있지만, 다음에 무엇을 해야 하는지 결정하는 구조화된 흐름이 부족한 경우가 많습니다.**

관찰된 사용자 니즈:

- 거친 앱 아이디어를 빠르게 실행 가능한 프로젝트 경로로 바꾸고 싶다.
- 사용자의 경험 수준에 따라 복잡도가 달라지는 체크리스트가 필요하다.
- 기획부터 배포까지 진행 상황을 한눈에 보고 싶다.
- AI 안내가 별도 채팅창에 떨어져 있지 않고 실제 작업 흐름 가까이에 있으면 좋다.
- 테스트, 남은 작업, 위험 요소 같은 중요한 단계를 놓치지 않았다는 확신이 필요하다.

## 4. 제품 접근 방식

VibePilot은 결정론적인 프로젝트 추적과 Copilot 기반 코칭을 결합합니다.

결정론적 레이어는 사용자에게 신뢰할 수 있는 기본 흐름을 제공합니다.

- 초보자, 중급자, 고급자 중 자신의 레벨을 선택한다.
- 프로젝트 이름, 목표, 시작일, 마감일을 입력한다.
- 선택한 레벨에 맞는 단계별 체크리스트를 받는다.
- 완료한 작업을 체크하며 진행 상황을 추적한다.
- 다음에 해야 할 미완료 작업을 확인한다.

AI 레이어는 상황에 맞는 생산성 지원을 더합니다.

- 사용자의 목표와 마감일을 바탕으로 프로젝트 로드맵을 다듬는다.
- 지금 가장 실용적인 다음 행동 하나를 추천한다.
- 각 작업이 왜 중요한지 설명한다.
- 현재 프로젝트가 다음 단계로 넘어갈 준비가 되었는지 점검한다.
- 사용자가 최종 선택권을 갖도록 체크리스트 개선안을 제안한다.

## 5. 현재 앱 기반

현재 리포지토리에는 사용자에게 보이는 MVP 기반이 이미 포함되어 있습니다.

| 영역 | 앱 동작 |
|---|---|
| 웹앱 | Next.js, React, TypeScript 기반 개인 생산성 앱 |
| 온보딩 | 초보자, 중급자, 고급자 레벨 선택 |
| 프로젝트 설정 | 프로젝트 이름, 목표, 시작일, 마감일 입력 |
| 기본 로드맵 | 기획부터 배포/운영까지 이어지는 레벨별 단계 |
| 진행 추적 | 체크리스트 토글, 단계별 진행률, 전체 진행률 |
| 동기부여 | 진행률에 따라 변화하는 마스코트 상태 |
| 다음 단계 표시 | 대시보드에서 다음 미완료 작업 안내 |

해커톤 버전은 이 기반 위에 Copilot SDK 안내를 핵심 생산성 기능으로 얹는 방향입니다.

## 6. 핵심 경험

1. 사용자는 VibePilot을 열고 바로 프로젝트 설정 흐름을 만납니다.
2. 사용자는 자신의 코딩 자신감에 맞는 레벨을 선택합니다.
3. 사용자는 프로젝트 이름, 목표, 시작일, 마감일을 입력합니다.
4. VibePilot은 레벨에 맞는 프로젝트 로드맵을 만듭니다.
5. 사용자는 AI 파일럿에게 로드맵 개선이나 다음 행동 추천을 요청합니다.
6. AI 응답은 가정, 판단 근거, 성공 기준과 함께 작업 흐름 안에서 스트리밍됩니다.
7. 사용자는 AI 제안을 적용하거나, 무시하거나, 다시 생성할 수 있습니다.
8. 사용자는 직접 진행 상황을 체크하며 계획의 주도권을 유지합니다.
9. 중요한 마일스톤 전에는 테스트, 남은 작업, 위험 요소, 최종 정리 상태를 점검합니다.

## 7. 기능 목표

| 기능 | 사용자 가치 | 확인 신호 |
|---|---|---|
| 레벨별 온보딩 | 부담을 줄이고 다양한 숙련도의 사용자에게 맞는 흐름을 제공한다. | 초보자, 중급자, 고급자마다 서로 다른 깊이의 로드맵이 생성된다. |
| 프로젝트 브리프 입력 | 막연한 아이디어를 구조화된 맥락으로 바꾼다. | 이름, 목표, 날짜, 레벨이 대시보드와 AI 흐름에 활용된다. |
| 진행 대시보드 | 사용자가 자신의 진행감을 안정적으로 확인한다. | 완료 체크가 단계 진행률, 전체 진행률, 다음 작업 표시를 갱신한다. |
| Copilot 로드맵 생성 | AI가 기획 워크플로의 중심 역할을 한다. | Copilot SDK를 통해 프로젝트 맥락에 맞는 로드맵을 생성할 수 있다. |
| 스트리밍 AI 응답 | AI가 빠르고 자연스럽게 통합된 느낌을 준다. | 전체 응답이 끝나기 전에도 유용한 텍스트가 화면에 나타난다. |
| 다음 행동 추천 | 사용자가 결정 피로 없이 바로 움직일 수 있게 한다. | AI가 하나의 추천 행동, 짧은 이유, 성공 기준을 제시한다. |
| 프로젝트 준비도 리뷰 | 아이디어에서 다음 마일스톤으로 넘어가는 순간을 돕는다. | 테스트, 남은 작업, 위험 요소, 사용자 목표 정렬 같은 실용적인 준비 항목을 짚어준다. |
| 제안 검토 | 사용자가 계획 변경의 주도권을 유지한다. | AI가 제안한 체크리스트 변경은 반영 전에 검토 상태로 표시된다. |

## 8. Copilot SDK 경험 설계

Copilot SDK는 VibePilot의 핵심 AI 레이어입니다. 앱은 사용자의 프로젝트 상태를 이해하고 구조화된 안내를 생성하는 프로젝트 코칭 세션을 위해 Copilot SDK를 활용합니다.

### 에이전트 콘셉트

**에이전트 이름:** VibePilot Coach

**에이전트 역할:** 1인 AI 보조 웹앱 프로젝트를 위한 실용적인 프로젝트 파일럿.

코치는 다음 방향을 지향합니다.

- 사용자가 선택한 숙련도 레벨을 존중한다.
- 긴 일반론보다 명확한 다음 행동 하나를 우선한다.
- 가정을 쉬운 말로 설명한다.
- 체크리스트 변경에서는 사용자의 선택권을 유지한다.
- 마감일, 미완료 작업, 프로젝트 준비 상태를 중요한 맥락으로 다룬다.

### AI에 전달되는 맥락

각 AI 요청은 다음과 같은 구조화된 프로젝트 맥락을 포함할 수 있습니다.

```json
{
  "projectName": "VibePilot",
  "goal": "바이브 코딩 프로젝트를 위한 개인 생산성 앱을 완성한다",
  "userLevel": "intermediate",
  "startDate": "2026-06-20",
  "endDate": "2026-06-20",
  "progressPercent": 35,
  "currentPhases": [
    {
      "id": "planning",
      "name": "기획",
      "completedSteps": 2,
      "totalSteps": 6
    }
  ],
  "submissionContext": {
    "webApp": true,
    "copilotSdk": true,
    "azureHosted": true,
    "modelLayer": "Azure OpenAI or Microsoft Foundry"
  }
}
```

### 제안되는 AI 도구

Copilot SDK 세션은 제품 흐름과 직접 연결되는 집중된 도구들을 노출할 수 있습니다.

| 도구 | 제품 내 목적 |
|---|---|
| `generateRoadmap` | 레벨에 맞는 로드맵을 만들거나 다듬는다. |
| `recommendNextAction` | 현재 상태에서 가장 유용한 다음 작업을 고른다. |
| `reviewProjectReadiness` | 현재 프로젝트가 다음 마일스톤으로 넘어갈 준비가 되었는지 점검한다. |
| `explainStep` | 선택한 체크리스트 항목의 가치를 설명한다. |
| `proposeChecklistPatch` | 사용자가 검토할 수 있는 로드맵 수정안을 제안한다. |

### 스트리밍 응답 형태

AI 응답은 빠르고 유용하게 느껴져야 합니다. 좋은 응답 흐름은 다음과 같습니다.

1. 현재 프로젝트 상태 진단.
2. 핵심 가정 또는 부족한 맥락.
3. 추천하는 다음 행동.
4. 선택적인 로드맵 조정안.
5. 구체적인 성공 기준.

이 구조는 AI 상호작용을 별도 대화가 아니라 작업 흐름에 자연스럽게 붙어 있는 생산성 기능처럼 느끼게 합니다.

## 9. UX 원칙

- 첫 화면은 바로 사용할 수 있는 앱 경험이어야 한다.
- 사용자는 문서를 읽지 않아도 다음에 무엇을 해야 하는지 알 수 있어야 한다.
- AI 안내는 실제 프로젝트 흐름 가까이에 나타나야 한다.
- 로딩, 스트리밍, 완료, 재시도 상태는 명확하게 보이면 좋다.
- AI를 사용할 수 없는 상황에서도 앱의 기본 흐름은 유용해야 한다.
- 초보자 모드는 단순하게, 고급자 모드는 기획부터 운영까지 더 넓은 흐름을 보여준다.
- 사용자는 AI 제안을 적용, 무시, 재생성할 수 있어야 한다.

## 10. 신뢰와 책임 있는 AI 원칙

VibePilot은 AI를 유용하게 만들되, 사용자가 이해하기 어려운 블랙박스로 만들지 않는 것을 지향합니다.

제품 신뢰 원칙:

- AI 제안은 자동 확정이 아니라 추천으로 제시된다.
- UI는 사용자가 확인한 진행 상황과 AI가 생성한 안내를 구분한다.
- 프로젝트 맥락이 부족할 때 AI는 가정을 드러낸다.
- 프롬프트에 시크릿을 입력하도록 유도하지 않는다.
- 모델 설정과 인증 정보는 서버 사이드에 둔다.
- 체크리스트 변경은 사용자가 검토할 수 있는 형태로 남긴다.
- 배포 안내는 준비를 돕는 역할이며, 실제 실행과 제출 결정은 사용자가 한다.

## 11. 데이터 모델

초기 제품 엔티티:

| 엔티티 | 필드 |
|---|---|
| Project | `id`, `name`, `goal`, `level`, `startDate`, `endDate`, `phases`, `repoUrl?` |
| Phase | `id`, `name`, `emoji`, `steps` |
| Step | `id`, `title`, `description`, `completed` |
| AiSuggestion | `id`, `type`, `summary`, `patch?`, `acceptanceCriteria`, `createdAt` |

해커톤 버전에서는 브라우저 로컬 저장만으로도 1인 사용자 흐름을 부드럽게 지원할 수 있습니다. 이후 프로덕션 버전에서는 인증과 데이터베이스 기반 프로젝트 저장을 추가할 수 있습니다.

## 12. 기술 방향

- AI 오케스트레이션은 서버 사이드에 두어 모델 설정이 브라우저 번들에 들어가지 않게 한다.
- 프롬프트 구성 로직은 UI 컴포넌트와 분리한다.
- 페이지 텍스트를 느슨하게 보내기보다 구조화된 요청 데이터를 AI 맥락으로 사용한다.
- 결정론적 체크리스트 상태와 AI 제안 상태를 분리한다.
- AI 흐름에 들어가는 사용자 입력을 검증한다.
- 진행률 계산, 요청 구성, 체크리스트 패치 처리 주변에 작은 테스트를 추가한다.
- AI 요청이 실패해도 프로젝트 진행 상황이 사라지지 않도록 친절한 오류 처리를 유지한다.
