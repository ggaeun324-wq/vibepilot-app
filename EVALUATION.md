# 평가 기준 요약 (총 100%)

| 순위 | 항목 | 가중치 | 핵심 키워드 |
|------|------|--------|-------------|
| 1 | Copilot SDK 활용 | 25% | 깊이 > 개수, 에이전트 설계, 도구 호출, 컨텍스트, 스트리밍 |
| 2 | 생산성 문제 해결 | 18% | 명확한 대상, 실질적 이점, 잘 정의된 문제 |
| 3 | Azure AI/Cloud | 18% | Foundry/Azure OpenAI 필수, 클라우드 네이티브 가점, 끼워넣기 감점 |
| 4 | 기술 실행력 | 16% | E2E 동작, 코드 품질, 아키텍처, 에러 처리 |
| 5 | UX/워크플로 | 12% | 낮은 저항, AI 자연스러운 통합, 사용자 주도권 |
| 6 | 책임 있는 AI | 6% | 투명성, human-in-the-loop, 프라이버시, 환각 완화 |
| 7 | 혁신성 | 5% | 뻔한 복제 X, 창의적 AI 적용 |

## 최고 점수 전략
- Copilot SDK를 앱 핵심 가치의 중심에 놓을 것 (깊게 1~2개 > 얕게 여러 개)
- AI 모델은 반드시 Azure OpenAI / Microsoft Foundry 위에서 동작
- 단순 CRUD가 아닌 AI가 생산성을 실질적으로 높이는 구조

# 코드 작성 규칙

1. **결정론적 성공 기준** — 각 기능의 성공/실패가 명확히 판별 가능해야 함
2. **의존성 완전 분리** — 기능별로 독립 모듈, 느슨한 결합
3. **테스트 최소선** — 핵심 로직에 최소 단위 테스트 포함
4. **입력값 검증** — 모든 외부 입력에 validation
5. **로그 & 에러 관측** — 구조화된 로깅, 에러 핸들링 + observability
